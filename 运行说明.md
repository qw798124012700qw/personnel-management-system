# 人事管理系统 —— Windows / VSCode 运行说明

本项目为《面向对象程序设计语言》课程设计，详细介绍见 `课程设计报告.docx`。
本说明只讲在 **Windows + VSCode** 下如何把代码跑起来。

## 一、环境（已就绪，无需另装）

代码使用本机已安装的 **NUAAProgEnv** 工具链（基于 MSYS2）：

| 工具 | 路径 |
| --- | --- |
| C++ 编译器 | `C:\Users\Public\NUAAProgEnv\msys64\clang64\bin\clang++.exe`（clang 21，C++17） |
| 调试器 | `C:\Users\Public\NUAAProgEnv\msys64\clang64\bin\gdb.exe` |
| make | `C:\Users\Public\NUAAProgEnv\msys64\usr\bin\make.exe` |
| Qt5 + qmake | `C:\Users\Public\NUAAProgEnv\msys64\clang64`（Qt 5.15.18，qmake 命令名为 `qmake-qt5`）|

VSCode 还需安装扩展 **C/C++**（`ms-vscode.cpptools`），用于智能提示与调试。

## 二、在 VSCode 中运行（推荐）

1. VSCode 打开文件夹 `11`（菜单 文件 → 打开文件夹）。
2. **编译**：按 `Ctrl+Shift+B`（默认任务 `build-console`），生成 `build/personnel_system.exe`。
3. **运行**：`Ctrl+Shift+P` → 输入 `Run Task` → 选择 `run-console`。
   程序会在集成终端启动（UTF‑8，中文正常显示，可直接键盘输入）。
4. **调试**：按 `F5`（配置「调试控制台版 (gdb)」），可设断点、单步执行。

> 工作目录已设为项目根目录，所以程序读写的 `data/employees.txt` 会落在 `11\data\` 下。

## 三、命令行运行（可选）

在项目根目录打开终端：

```sh
make          # 编译
make run      # 编译并运行
make clean    # 清除可执行文件
```

或直接调用编译器：

```sh
clang++ -std=c++17 -Wall -Wextra -pedantic src/main.cpp src/personnel_system.cpp -o build/personnel_system.exe
./build/personnel_system.exe
```

## 四、Qt 图形界面版（qt_gui）

已安装 **Qt 5.15.18**，图形界面版可正常构建运行。

**在 VSCode 中**：
- 构建：`Run Task` → `build-gui`
- 运行：`Run Task` → `run-gui`（会先构建再启动窗口）
- 调试/F5：选择配置「运行 Qt 图形界面版 (gdb)」

**命令行**（项目根目录）：

```sh
make gui        # 构建 -> qt_gui/release/personnel_gui.exe
make run-gui    # 构建并运行
make clean-gui  # 清除图形界面版构建产物
```

底层等价于在 `qt_gui` 目录执行 `qmake-qt5 personnel_gui.pro && mingw32-make`，
然后把平台插件 `qwindows.dll` 复制到 `release/platforms/`。

> 说明：
> - 运行时所需的 Qt5 DLL 位于 `clang64\bin`（已在系统 PATH 上），无需额外复制。
> - 图形界面版以 `qt_gui` 为工作目录运行，其数据路径 `../data/employees.txt`
>   正好指向项目根的 `data/`，**与控制台版共用同一份数据**。
