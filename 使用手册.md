# 人事管理系统 · 使用手册

本手册详细说明**控制台版**和**图形界面版**的全部功能与操作步骤,适合第一次使用者照着操作。

---

## 一、系统简介

人事管理系统用于集中管理员工信息,支持员工的**录入、查询、修改、删除、排序、统计、文件保存/读取**。系统提供两个版本,功能一致、数据互通:

- **控制台版**:命令行菜单操作,输入数字选择功能。
- **图形界面版(Qt)**:窗口、表格、表单、按钮操作,更直观。

两个版本**共用同一个数据库** `data/employees.db`(SQLite),在一个版本里保存后,另一个版本读取即可看到。

---

## 二、启动方法

下载并解压发布包(Releases 里的免安装 zip)后,文件夹内:

| 版本 | 启动方式 |
| --- | --- |
| 图形界面版 | 双击 **`人事管理系统.exe`** |
| 控制台版 | 双击 **`人事管理系统-控制台版.exe`** |

> 程序启动时会自动从数据库读取已有员工;首次运行(数据库为空)会自动从 `data\employees.txt` 种子数据迁移。

---

## 三、数据存储说明

- 运行数据保存在 SQLite 数据库 `data\employees.db`(两个版本共用)。
- **种子数据**:首次运行(数据库为空)会自动从同目录的 `data\employees.txt` 迁移并写回数据库。该文本文件为 UTF‑8,每个员工占一行,10 个字段用竖线 `|` 分隔,顺序为:
  `姓名|性别|身份证号|生日|电话|工作证号|家庭地址|薪水|职务|部门`
- 读取时若某行字段数不对、日期非法、薪水非数字或**工作证号重复**,该行会被**自动跳过**并提示,不影响其它记录。
- 一般不需要手工编辑该文件,通过程序的"保存"功能写入即可。

---

## 四、字段与输入规则(重要)

录入或修改时,各字段的要求如下:

| 字段 | 规则 / 示例 |
| --- | --- |
| 姓名 | 不能为空。例:张三 |
| 性别 | 只能是 **男** 或 **女** |
| 身份证号码 | **15 位或 18 位**,18 位末位可为 X。例:110105199001011234 |
| 生日 | 格式 **YYYY-MM-DD**,必须是真实日期(自动判断闰年,如 2024-02-29 合法、2023-02-29 非法) |
| 电话号码 | **7–15 位**,只能含数字和短横线 `-`。例:13800138000 |
| 工作证号 | 不能为空,且**不能与已有员工重复**(系统的唯一标识) |
| 部门 | 不能为空。例:研发部 |
| 职务 | 不能为空。例:软件工程师 |
| 薪水 | **非负数字**,可带一位小数。例:8500 或 8500.50 |
| 家庭地址 | 不能为空 |

> 输入不合法时,程序会提示原因并要求重新输入(控制台版)或弹窗警告(图形界面版)。

## 五、控制台版使用说明

启动后显示主菜单,在"请选择: "后**输入对应数字并回车**:

```
========== 人事管理系统 ==========
1. 添加员工信息    2. 删除员工信息    3. 清空全部员工信息
4. 显示全部员工信息  5. 查找员工信息    6. 修改员工信息
7. 保存员工信息    8. 读取员工信息    9. 高级查询
10. 排序管理      11. 统计分析       0. 退出系统
```

**1 添加员工信息**
按提示**依次**录入:姓名 → 性别 → 身份证号 → 生日 → 电话 → 工作证号 → 部门 → 职务 → 薪水 → 家庭地址。
每项都会校验格式(见第四节);工作证号若已存在,提示"工作证号已存在,添加失败"。

**2 删除员工信息**
输入要删除的**姓名或工作证号** → 若匹配到多条,按序号选择一条(输入 0 取消) → 确认 `y/n`。

**3 清空全部员工信息**
删除全部记录,**需输入 y 确认**(不可撤销),输入 n 取消。

**4 显示全部员工信息**
以表格列出全部员工的:序号、姓名、身份证号、工作证号、生日、部门。

**5 查找员工信息**
输入**姓名或工作证号**:姓名**模糊匹配**(输入"张"可匹配"张三"),工作证号**精确匹配**。显示所有匹配员工的**详细**信息。

**6 修改员工信息**
输入姓名或工作证号选定员工(多条则选序号)→ 进入二级菜单,输入 **1–10** 选择要修改的项 → 输入新值 → 可继续改其它项 → 输入 **0** 返回。
若改后的工作证号与他人重复,系统会自动恢复原工作证号并提示。

**7 保存员工信息**
手动把全部员工**写入数据库** `data\employees.db`。注意:增删改已**实时写库**,本项相当于再确认保存一次。

**8 读取员工信息**
从数据库 `data\employees.db` **重新载入**(会覆盖当前内存中的数据)。

**9 高级查询**(子菜单,均由数据库 SQL 查询完成)
- `1` 按部门查询:输入部门名,**精确**列出该部门所有员工(SQL `WHERE department = ?`)。
- `2` 按薪水区间查询:依次输入最低、最高薪水(若写反会自动交换),列出区间内员工(SQL `BETWEEN`)。
- `3` 按职务关键字查询:输入关键字,**模糊**匹配(SQL `LIKE`,输入"工程"可匹配"软件工程师")。
- `0` 返回主菜单。

**10 排序管理**(子菜单)
- `1` 按工作证号升序　`2` 按生日升序(越早越前)　`3` 按薪水降序(越高越前)　`0` 返回。
- 排序后立即显示结果。

**11 统计分析**(子菜单)
- `1` 按部门统计人数　`2` 薪水统计(总额、平均、最高/最低薪水及对应姓名)　`0` 返回。

**0 退出系统**
直接退出。增删改已实时写入数据库,无需在退出时手动保存。

> 提示:控制台版以数据库为实时数据源,每次增 / 删 / 改都会立即写入 `data\employees.db`,不会因忘记保存而丢失。

## 六、图形界面版使用说明

主窗口从上到下分为几个区域:

**① 顶部标题栏**:显示系统名称,右侧徽章显示当前**记录条数**。

**② 查询与筛选区**(左上):
- "查询方式"下拉:可选 *全部字段关键字 / 姓名 / 部门 / 职务 / 工作证号*。
- "输入关键字"框:**边输入边即时筛选**表格,无需点按钮。
- "清除筛选"按钮:清空关键字、恢复显示全部。
- "薪水区间查询"按钮:弹窗依次输入最低、最高薪水,结果以弹窗列出。

**③ 中部员工表格**:显示全部员工(10 列)。
- **单击某一行**:该员工信息自动填入下方表单,便于修改。
- **单击列标题**:按该列排序(再次点击切换升/降序)。

**④ 下方"员工信息"表单**:用于录入/编辑。
- 姓名、身份证、电话、工作证号、部门、职务、家庭地址:文本框。
- 性别:下拉选择(男/女)。
- 生日:日期选择框(可点日历)。
- 薪水:数字框(带 ¥ 前缀,可调小数)。

**⑤ "操作"按钮区**:
| 按钮 | 作用 |
| --- | --- |
| 添加 | 把表单内容新增为一名员工(校验后) |
| 修改 | 把表单内容更新到**当前选中行**的员工 |
| 删除 | 删除**当前选中行**的员工(弹窗确认) |
| 清空表单 | 清空表单内容,便于重新录入 |
| 读取文件 | 从数据文件重新载入 |
| 保存文件 | 把当前数据写入数据文件 |
| 部门统计 | 弹窗显示各部门人数 |
| 薪水统计 | 弹窗显示总额/平均/最高/最低 |
| 按工号排序 | 按工作证号升序 |
| 按薪水排序 | 按薪水降序 |
| 导出CSV | 把当前数据导出为 CSV 文件(可用 Excel / WPS 打开) |
| 撤销 | 多级回退最近的增 / 删 / 改操作(快捷键 `Ctrl+Z`;无可撤销时为灰色) |
| 重做 | 恢复被撤销的操作(快捷键 `Ctrl+Y` 或 `Ctrl+Shift+Z`;无可重做时为灰色) |

**⑥ 底部状态栏**:显示操作结果提示(如"添加成功""当前共有 N 条")。

**典型操作:**
- **添加**:在表单填好各项 → 点「添加」。
- **修改**:在表格点选要改的员工(信息会回填表单)→ 修改表单 → 点「修改」。
- **删除**:在表格点选员工 → 点「删除」→ 确认。
- **保存**:做完增删改后**点「保存文件」**写入磁盘。

> **重要**:关闭窗口时若有未保存的改动,程序会弹窗询问"是否保存后退出?"——选「保存」可避免数据丢失。
>
> **列宽与排序**:拖动列边界可调整列宽、点列头可排序;这些状态会在关闭时记住,下次启动自动恢复。

## 七、典型操作流程示例

**例:新增一名员工并保存**
1. 启动程序(已自动加载现有员工)。
2. 控制台版:输入 `1`,依次录入各字段;图形界面版:在表单填好后点「添加」。
3. 检查:控制台版输入 `4` 显示列表;图形界面版表格中可见新员工。
4. 保存:控制台版输入 `7`;图形界面版点「保存文件」。

**例:查询研发部、薪水 8000–12000 的人**
- 控制台版:`9` → `1` 输入 `研发部`;再 `9` → `2` 输入 `8000`、`12000`。
- 图形界面版:查询方式选「部门」输入 `研发部`;或点「薪水区间查询」输入 8000、12000。

## 八、常见问题与注意事项

- **控制台中文乱码?** 已在程序内将控制台切换为 UTF‑8,正常不会乱码;如仍异常,在终端(PowerShell)里运行该 exe 查看输出。
- **改了数据没保存就关了?** 控制台版退出时会问是否保存;图形界面版关窗口时会提示——务必选保存。
- **两个版本数据不同步?** 它们共用 `data\employees.db`;在一个版本保存后,到另一个版本点「读取文件」/选 `8` 即可同步。
- **工作证号必须唯一**:重复的工作证号无法添加,导入时也会被跳过。
- **便携版迁移**:把整个解压后的文件夹一起拷贝,不要只拷 exe(图形界面版需要同目录的 DLL 以及 `platforms`、`sqldrivers` 文件夹)。
